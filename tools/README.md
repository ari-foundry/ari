# Tools

Developer tools built around the Ari compiler live here. Keep reusable Python support under `tools/ari_tooling/` and user-facing commands in their own tool folders.

README files are useful for tool boundaries such as `tools/lint/`,
`tools/lsp/`, and `tools/ari_tooling/`, where a contributor needs setup or
ownership notes before editing. Keep smaller fixture/output folders documented
from their parent instead of adding a README to every directory.
